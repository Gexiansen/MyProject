# HTML_Project 项目规范

## 项目定位

- 这是一个位于 `docs/index.html` 的单文件家庭财务应用。
- 页面使用 React、Recharts、Tailwind CSS 和 Babel 的 CDN 版本，不引入本地构建流程。
- 默认保持单文件部署方式，除非用户明确批准拆分或引入构建工具。

## 修改边界

- 只修改与当前需求直接相关的代码，不顺手重构或格式化无关区域。
- 保持现有 `accounts`、`records`、`withdrawalRate` 数据结构兼容。
- 不改变财务计算公式，除非需求明确要求并先讨论计算口径。
- 浏览器导出的财务 JSON 属于用户本地数据，默认不修改、不提交 Git；数据处理任务除外，且必须保留原文件。
- 密码功能只是本机页面锁，不将其描述为数据加密或服务端认证。

## UI 与交互

- 默认优先适配手机，同时保证桌面端可用。
- 新增交互必须延续现有自动保存方式，不额外引入手动保存流程。
- 涉及删除或覆盖财务数据的操作必须有明确确认提示。
- 显示成员时使用账户的 `member` 字段，不把成员文本写入账户名称。

## 验证要求

- 修改后运行 `git diff --check`。
- 涉及页面渲染、图表或交互时，使用本地 HTTP 服务在浏览器中验证。
- 至少检查桌面宽度和 390px 手机宽度，确认无横向溢出和控制台错误。
- 涉及数据计算时覆盖空值、零值、负值及历史月份场景。

## 进度维护

- `ROADMAP.md` 是项目真实进度源。
- 只有完成实现并通过验证的事项才能写入“已完成”。
- 开发或修复完成后必须同步更新 `ROADMAP.md`。

## Git

- 日常开发使用 `main` 分支。
- Commit 信息使用中文描述和 `feat:`、`fix:`、`docs:`、`refactor:`、`chore:` 前缀。
- 不提交财务导出 JSON，除非用户明确要求。
